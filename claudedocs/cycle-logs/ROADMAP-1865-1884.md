# Roadmap: Cycles 1865-1884 — Performance Improvement Focus

> **Scope**: 성능 향상에 집중. 골든테스트/벤치마크 추가 금지.
> **Starting state**: v0.96.45, codegen NEAR-OPTIMAL, 120+ FASTER, 1 FAIL 1 WARN

## Phase A: Runtime Modularization (Cycles 1865-1870)
Split monolithic bmb_runtime.c (5,759 LOC, 381 functions) into modules:
- bmb_core.c: print, assert, panic, abs, min, max, conversions (~100 LOC)
- bmb_string.c: string operations (~600 LOC)
- bmb_vec.c: vector operations (~400 LOC)
- bmb_map.c: hashmap operations (~400 LOC)
- bmb_concurrent.c: threading, channels, mutexes, atomics (~1500 LOC)
- bmb_arena.c: arena allocator (~300 LOC)
- bmb_io.c: file I/O, sockets, process (~800 LOC)
- bmb_runtime.c: remaining + includes
Update build pipeline to compile each module separately.
Measure binary size and benchmark impact.

## Phase B: MIR Type Propagation (Cycles 1871-1876)
Add type annotations to bootstrap MIR to enable:
- nonnull/noalias on String parameters (currently lost because MIR uses i64)
- Type-specific optimizations
- Better alias analysis in LLVM

## Phase C: Bootstrap SAE + Contract Optimizations (Cycles 1877-1882)
- Port SaturatingArithElimination to bootstrap/optimize.bmb
- Contract-based dead branch elimination in bootstrap
- range() postcondition support

## Phase D: Review + Polish (Cycles 1883-1884)
- Full verification (tests + bootstrap + benchmarks)
- Profile compiler hot paths for speed
- Document improvements
