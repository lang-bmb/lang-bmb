### Bug Found + Fixed: f64/bool in Generic Enum Native Compilation
**Symptom**: `Option<f64>` and `Option<bool>` compiled but produced wrong results or type errors in LLVM IR.

**Root cause**: Enum slots are always i64. When monomorphized generic function (e.g., `get_or_f64`) extracts a match binding from an enum, the value is i64 but the phi/return expects double (or i1 for bool).

**Fix 1** (phi coercion detection): `llvm_text.rs:2152` — Added `i64→double`, `double→i64`, `i64→i1`, `i64→i32` to phi coercion detection

**Fix 2** (return type conversion): `llvm_text.rs:7188` — When phi produces i64 but function returns double, bitcast at ret. When returns i1, trunc at ret.

**Fix 3** (coercion instruction): `llvm_text.rs:2469` — Added `i64→i1` trunc case
