### IR 검증

```llvm
; @pure @noinline → noinline + memory(none) + speculatable
define private i64 @expensive(i64 %x) noinline nosync nounwind willreturn mustprogress
  nofree norecurse memory(none) speculatable {
```

### LLVM 최적화 검증
- 3회 호출 → **1회** (GVN이 memory(none)으로 CSE 수행, 함수 body 미참조)
- `expensive(x) * 3` 패턴으로 최적화 (mul nsw 변환)
- 단순 상수 인자: 컴파일 타임 평가 (speculatable)

### 벤치마크 결과 (purity_opt @noinline)

| Runtime | Median (ms) | vs BMB |
|---------|-------------|--------|
| **BMB (@pure @noinline)** | **121** | **1.00x** |
| C (Clang -O3 __noinline__) | 344 | **2.84x slower** |
| C (GCC -O3 __noinline__) | 586 | **4.84x slower** |

**결론**: @noinline에서도 @pure의 성능 우위 동일. LLVM은 memory(none)으로 함수 body 없이 최적화 가능. C는 noinline 함수의 순수성을 알 수 없어 최적화 불가.
